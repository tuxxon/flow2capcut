### 4단계: 시놉시스 작성 ⭐NEW

**`야담_시놉시스_작성_지침.md`를 반드시 읽고 시작한다.**

20챕터 프레임워크에 따라 시놉시스를 작성한다.

#### 20챕터 구조

```
【기】 도입부 (1~5)
├─ 1. 시대적 배경 설정
├─ 2. 주인공 소개
├─ 3. 주변 인물 관계망
├─ 4. 평화로운 일상과 변화의 조짐
└─ 5. 사건 발생 — 일상의 균열

【승】 전개부 (6~12)
├─ 6. 첫 번째 선택과 행동
├─ 7. 이야기 흐름을 바꾸는 존재 등장
├─ 8. 첫 번째 시련과 대응
├─ 9. 첫 번째 반전 — 상황의 변화
├─ 10. 관계의 심화와 변화
├─ 11. 두 번째 시련 — 위기의 심화
└─ 12. 중요한 비밀/정보 발견

【전】 위기부 (13~17)
├─ 13. 주인공의 내적 성장과 변화
├─ 14. 핵심 반전 — 상황 역전
├─ 15. 최대 위기 — 절망의 순간
├─ 16. 결정적 선택과 행동
└─ 17. 절정 — 최종 대결/시험

【결】 결말부 (18~20)
├─ 18. 모든 것의 진실 드러남
├─ 19. 갈등 해소와 새로운 질서
└─ 20. 교훈과 여운
```

#### 시놉시스에서 미리 설계할 것

- **🔴 궁금증 유지 설계 (최우선)**: 시놉시스 단계에서 "시청자가 범인/진실을 확신하는 최초 지점"을 명시하고, 그것이 15챕터(전 중반) 이후인지 반드시 확인
- **감정-행동 매핑**: 주요 감정 장면마다 Show 방법 기획
- **서스펜스 기법 배치**: 어느 챕터에 어떤 기법을 쓸지 설계
  - 기(1~5): 오픈 루프, 정보 격차, 시드 배치
  - 승(6~12): 극적 아이러니, 점진적 해빙, 에스컬레이션
  - 전(13~17): 클리프행어, 타임 프레셔, 거짓 해결, 패턴 파괴
  - 결(18~20): 콜백, 아이러니 역전, 작은 은혜의 복리
- **복선 추적표 초안**: 설치/회수 챕터 미리 설정
- **의심 대상 배치**: 진범 외 최소 2명의 미끼 용의자 설계
- **🔴 반전 갭 설계**: 진범을 시청자가 가장 신뢰하는 인물로 위장 → 폭로 시 갭 극대화. 반전 전까지 신뢰를 쌓는 장면을 충분히 배치
- **시네마틱 포인트**: 슬로모션, 병렬 편집 등 핵심 장면 설계

**출력 파일**: `04_시놉시스.md`

### 6단계: 대본(시나리오) 작성

**`야담_시나리오_작성_지침.md`, `야담_서술기법_가이드.md`, `야담_서스펜스_기법.md`를 반드시 읽고 시작한다.**

시놉시스를 바탕으로 **훅 + 기승전결 4파트를 모두 작성한 다음, 7단계에서 통합 검토**한다.

#### 작성 순서

```
훅(오프닝) → 기 → 승 → 전 → 결 → (모두 완성 후) 7단계 검토
```

#### 파일 구조

```
ep{번호}/
├── 01_분석.md
├── 02_팩트체크.md
├── 03_자료수집.md
├── 04_시놉시스.md
├── 05_프리플라이트.md
├── {제목}_기.md           ← 도입부
├── {제목}_승.md           ← 전개부
├── {제목}_전.md           ← 위기/반전
├── {제목}_결.md           ← 결말
├── 07_검토.md
├── references.csv          ← 인물/장소/스타일 레퍼런스
├── {제목}_scenes.csv       ← 씬별 데이터 (프롬프트, 자막, 타임코드)
├── 08_sfx_목록.md          ← SFX 추출 목록
├── segments_{파트}/          ← 세그먼트별 mp3 + JSON 타임스탬프

├── final_{파트}.srt          ← 파트별 자막 (의미 단위)
├── sfx/                     ← SFX 원본 (파트별 MMSS 타임코드)
├── generate_tts.py          ← TTS 생성 스크립트 (with-timestamps)
├── generate_srt.py          ← SRT 생성 스크립트 (수동 자막 기반)

├── sfx_timecodes.py         ← SFX 타임코드 매핑 참조
├── media/                   ← 최종 프로덕션 산출물
│   ├── final_full.mp3       ← 전체 병합 오디오 (기+승+전+결)
│   ├── final_full.srt       ← 전체 병합 자막 (오프셋 적용)
│   └── sfx/                 ← SFX (전체 타임라인 기준 MMSS)
│       ├── 01_주판_구슬_튕기기_0030.mp3
│       ├── 13_시장_소리_0836.mp3      ← 승 2:01 → 전체 8:36

#### 글자수 배분 (적정 범위: 8,000 ~ 12,000자)

| 구간 | 글자 수 | 비율 |
|------|---------|------|
| 【기】 도입부 | 1,500자 | 15% |
| 【승】 전개부 | 3,000자 | 30% |
| 【전】 위기부 | 3,500자 | 35% |
| 【결】 결말부 | 2,000자 | 20% |
| **합계** | **10,000자** | 100% |

> 글자수 확인: `LC_ALL=ko_KR.UTF-8 wc -m`

#### 문체 규칙

- **시점**: 3인칭 관찰자 시점 일관 유지
- **종결**: "~습니다", "~지요" 존댓말 형식
- **시작**: "옛날 옛적" 또는 "먼 옛날"
- **마무리 CTA**: "다음 영상을 빠르게 만나 보시려면 좋아요와 구독을 눌러 주세요. 그럼 더 재미있는 영상으로 찾아뵙겠습니다. 모두 행복한 하루 보내세요."
- **대사**: 자연스러운 사극 말투 ("~하옵니다", "~이로구나")
- **신분별 화법**: 양반→서민 반말, 서민→양반 극존칭

#### Show, Don't Tell 핵심 규칙

감정을 직접 명명하지 않고 행동/신체반응/환경/대사로 보여준다.

| 감정 | ❌ Tell | ⭕ Show |
|------|--------|--------|
| 슬픔 | "슬펐습니다" | 고개를 숙이다, 입술을 깨물다, 눈시울이 붉어지다 |
| 분노 | "분노했습니다" | 주먹을 불끈 쥐다, 이를 악물다, 목소리가 떨리다 |
| 공포 | "두려웠습니다" | 등줄기에 한기가 흐르다, 다리가 굳다 |
| 사랑 | "사랑했습니다" | 볼이 붉어지다, 눈을 피하다, 심장이 뛰다 |
| 감동 | "감동했습니다" | 눈시울이 뜨거워지다, 목이 메다 |

#### 서스펜스 기법 (대본 적용)

| 기법 | 적용 위치 |
|------|-----------|
| 오픈 루프 | 기~승 초반, 질문을 던지고 답을 2~3챕터 뒤에 |
| 극적 아이러니 | 승, 시청자는 아는데 인물은 모르는 상황 |
| 타임 프레셔 | 전, 시간 제한으로 긴박감 |
| 거짓 해결 | 전, 해결된 것 같다가 더 큰 문제 |
| 콜백 | 결, 초반 대사/장면의 의미 변화 |

#### 정보 공개 원칙 — 조금씩 풀기

나레이터가 친절하게 해설하면 시청자의 추론 기회를 빼앗는다. **정보는 일부러 덜 알려준다.**

| ❌ 친절한 해설 (나쁜 예) | ⭕ 조금씩 풀기 (좋은 예) |
|------------------------|----------------------|
| "최씨가 조작한 것이 아니었습니다. 누군가의 조작을 감지하고 혼자서 기록해두고 있었다는 뜻이었지요." | "최씨 장부에는 원래 금액이 적혀 있었지요. 소은의 손이 멈추었습니다." |
| "소은은 그때 아무 이상함도 느끼지 못했습니다." (의도 해설) | 질문 대사만 남기고, 시청자가 눈치채게 두기 |
| "이서방이 전부가 아니었습니다." (결론을 대사로 말함) | 차액이 다시 나기 시작 → 소은의 입술이 굳는 행동만 |
| 격언/교훈을 여러 번 반복 ("실수는 고르지 않고, 의도는 고르다") | 격언은 설치 1회 + 회수 1회만. 중간 반복 금지 |

**핵심**: 나레이터는 사실만 전달하고, **의미/해석은 시청자가 추론**하게 한다. "뭔가 이상했다" → 나중에 공개 ✅. 바로 설명 ❌.

#### 해결/액션 장면 원칙 — 빠르고 강하게

위기 해결, 구출, 대결 장면은 **서술 설명을 줄이고 액션 중심**으로 쓴다.

| ❌ 서술형 (느린 예) | ⭕ 액션형 (빠른 예) |
|-------------------|-------------------|
| "최씨에게 곳간 열쇠를 구했으나, 곽 주사가 미리 빼돌린 뒤였습니다." | "열쇠가 없었습니다. 복돌이가 쇠지렛대를 끌고 왔지요. 빗장이 비틀어졌습니다." |
| "소은이 어둠 속에서 장부를 다시 한번 자세히 살펴보았습니다." | "소은이 장부를 펼쳤습니다. 주판을 무릎에 올렸지요. 달그락, 달그락." |
| 화해/고백 대사가 3~4문장 | 핵심 대사 1~2문장 + 행동/신체반응 |

**핵심**: 위기→해결 구간은 **짧은 문장 + 동사 중심 + 소리/움직임**으로 템포를 올린다. 설명이 들어가면 긴장이 풀린다.

#### 도입부(기) — 첫 30초가 승부

- 첫 문장에서 강렬한 상황을 던진다
- "어느 날" 같은 평범한 시작 금지
- 3문장 안에 핵심 갈등이나 미스터리 암시

#### 전개부(승) — 궁금증을 쌓아올린다

- 의심 대상 최소 2명 배치 (미스터리 구조)
- 작은 승리 → 더 큰 위기 패턴 반복
- 진범을 아군/조력자로 위장
- 감정적 유대 형성 장면 (인정, 우정, 효심)

#### 위기/반전(전) — 반전은 2중으로, 갭은 극대로

- 1차 반전: "설마?" + 2차 반전: "이것까지?!"
- **🔴 범인 정체는 전 중반(15챕터) 이후에 폭로** — 그 이전에 확정되면 구조적 실패
- **🔴 반전 갭 원칙**: 시청자가 믿고 있던 것과 진실 사이의 거리(갭)가 클수록 좋다
  - 가장 믿었던 아군이 진범 → 갭 극대 → 충격 극대
  - 가장 의심했던 인물이 사실 보호자 → 이중 반전
  - 반전 전에 그 인물을 시청자가 충분히 신뢰/의심하도록 장면을 쌓아야 갭이 커진다
- 반전 후 감정의 깊이를 주는 장면 (충격 → 배신감 → 분노 → 카타르시스)

#### 결말(결) — 여운을 남긴다

- 복선 회수로 쾌감
- 콜백으로 감동 극대화
- 교훈과 여운을 담는 마지막 문장

#### 매 챕터 전환 시 체크

- [ ] 감정을 직접 명명한 곳이 없는가? (Show, Don't Tell)
- [ ] 시네마틱 기법 1개 이상 활용했는가?
- [ ] 오감 중 2가지 이상 활용했는가?
- [ ] 이전 챕터와 자연스럽게 연결되는가?
- [ ] 글자수 배분을 준수했는가?
- [ ] 복선 추적표가 최신 상태인가?

### 7단계: 검토 후 수정 (반복)

작성된 대본을 자체 검토 → subagent 재검토 → 수정을 **수정사항이 없을 때까지 반복**한다. (최대 5라운드)

```
┌─ 1. 자체 검토 + 수정사항 목록 작성
│     ▼
│  수정사항 있음? → YES → 대본 수정
│                         ▼
│                  subagent 재검토 (대본 파일 직접 읽기)
│                         ▼
│                  수정사항 있음? → YES → 수정 → 재검토 (반복)
