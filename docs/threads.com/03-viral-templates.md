## Template 1: Transformations (변환 스토리)

> 변화를 보여주면 사람들이 주목한다. Before → After의 차이가 클수록 좋다.

**구조**:
```
[Before 상태 — 고통/문제]
↓
[전환점 — 깨달음/행동]
↓
[After 상태 — 결과]
```

**일반 예제**:
```
6 months ago: spending 12 hours/week on manual social media posting.
today: everything is automated. i post once and it goes everywhere.

here's the one tool that changed everything for me.
```

**Whisk2CapCut EN**:
```
3 months ago:
- downloading AI images one by one
- dragging 200+ files into CapCut manually
- adding subtitles frame by frame
- 3 full days per video project

today:
- one click
- 200 images → CapCut project in 10 minutes
- subtitles, animations, everything included

i built the tool that made this possible.
```

**Whisk2CapCut KO**:
```
3개월 전:
- AI 이미지 하나씩 다운로드
- 200장+ 파일을 CapCut에 수동으로 끌어다 놓기
- 자막을 프레임별로 수동 입력
- 영상 하나에 3일

지금:
- 클릭 한 번
- 200장 → CapCut 프로젝트 10분
- 자막, 애니메이션 전부 포함

이걸 가능하게 한 도구를 직접 만들었어요.
```

## Template 2: Call People Out (특정 대상 지목)

> 대상을 특정하면 "이거 내 얘기다" 싶어서 답글이 달린다.

**구조**:
```
[특정 대상]에게 고함 —
[그들의 구체적 고통]
[공감 → 해결 암시]
```

**일반 예제**:
```
calling out every solo developer who has 47 browser tabs open right now.

you don't need another tutorial.
you need to ship.
```

**Whisk2CapCut EN**:
```
this is for every AI video creator who's still dragging images into CapCut one by one.

i see you.
i was you.

3 nights of copy-paste hell later, i decided to automate the entire thing.

if you make AI slideshows or videos — we should talk.
```

**Whisk2CapCut KO**:
```
아직도 이미지를 CapCut에 하나씩 끌어다 놓는 AI 영상 크리에이터분들 —

저도 그랬어요.
3일 밤을 복사-붙여넣기 지옥에서 보내고 나서
전부 자동화하기로 했어요.

AI 슬라이드쇼나 영상 만드는 분이면 같이 이야기해요.
```

## Template 3: Share Your Lessons (실패 교훈)

> 실패 → 교훈이 가장 강력한 스토리텔링. "왜 들어야 하는지"를 먼저 말해라.

**구조**:
```
[왜 내 말을 들어야 하는지 — 실패/경험]
[배운 교훈 리스트]
[핵심 한 줄]
```

**일반 예제**:
```
i failed at 3 startups before one worked.

here's what each failure taught me:
1. first one: built what i wanted, not what users needed
2. second one: launched too late, market moved on
3. third one: no distribution, great product nobody saw

lesson: distribution > product. always.
```

**Whisk2CapCut EN**:
```
i built 3 versions of my automation tool before one actually worked.

version 1: tried to do everything → too complex, nobody used it
version 2: chrome extension only → limited by browser sandbox
version 3: desktop app + simple UI → finally clicked

what i learned:
- solve ONE painful step, not the whole pipeline
- desktop beats browser for heavy automation
- if it takes more than 2 clicks, you lost the user
```

**Whisk2CapCut KO**:
```
자동화 도구를 3번 만들었는데 3번 다 실패했어요.

버전 1: 모든 걸 다 하려고 함 → 너무 복잡, 아무도 안 씀
버전 2: 크롬 확장만 → 브라우저 제한에 막힘
버전 3: 데스크톱 앱 + 단순 UI → 드디어 성공

배운 점:
- 전체 파이프라인이 아니라 가장 아픈 한 단계만 해결하기
- 무거운 자동화는 브라우저보다 데스크톱이 낫다
- 클릭 2번 넘으면 유저가 떠난다
```

## Template 4: Share Your Controversy (논쟁적 의견)

> 진심으로 믿는 바를 말하면 진정성이 느껴지고, 동의/반대 양쪽에서 답글이 온다.

**구조**:
```
[논쟁적 주장]
[근거 — 왜 그렇게 생각하는지]
[열린 질문으로 마무리]
```

**Whisk2CapCut EN**:
```
hot take: if you're still manually adding images to your video editor in 2026, you're working against yourself.

the technology to automate this has existed for over a year.
most people just don't know it exists.

or worse — they think "automation = low quality."
it's the opposite. automation frees you to focus on the creative part.

am i wrong?
```

**Whisk2CapCut KO**:
```
좀 과격한 말일 수 있는데 — 2026년에 아직도 편집기에 이미지를 수동으로 넣고 있다면, 자기 자신과 싸우는 거예요.

이걸 자동화하는 기술은 1년 전부터 있었어요.
대부분 존재 자체를 모르거든요.

아니면 더 심한 건 — "자동화 = 낮은 퀄리티"라고 생각하는 거.
정반대예요. 자동화가 있어야 창작에 집중할 수 있어요.

제가 틀렸나요?
```

## Template 5: "Took Me X → You Get It in Y" (압축 가치)

> 오래 걸린 경험을 짧게 압축 → 저장(bookmark) 유도 → 알고리즘 부스트.

**구조**:
```
[나는 X 시간/년 걸렸다]
[당신은 Y 분이면 된다]
[압축된 가치 리스트]
```

**Whisk2CapCut EN**:
```
took me 6 months of trial and error to build the perfect AI video workflow.

you get it in 60 seconds:

1. write script
2. TTS → voice + SRT subtitles
3. split scenes from SRT timestamps
4. write image prompts per scene
5. bulk generate 200+ images (Google Whisk)
6. one-click export → CapCut project
7. final edit in CapCut → publish

the bottleneck was always steps 5-6.
3 days of manual work → now 10 minutes.
```

**Whisk2CapCut KO**:
```
완벽한 AI 영상 워크플로우를 찾는 데 6개월 걸렸어요.

60초면 알려드릴게요:

1. 대본 작성
2. TTS → 음성 + SRT 자막
3. SRT 타임스탬프로 씬 분할
4. 씬별 이미지 프롬프트
5. AI 이미지 200장+ 대량 생성 (Google Whisk)
6. 원클릭 → CapCut 프로젝트
7. CapCut에서 최종 편집 → 업로드

병목은 항상 5~6번이었어요.
수작업 3일 → 이제 10분.
```

## Template 6: Before-Path-After (검증된 스토리 공식)

> Before와 After의 대비를 크게, 사이에 Path(여정)를 넣는다.

**구조**:
```
Before: [고통스러운 상태]
Path: [무엇을 했는지]
After: [변화된 상태]
```

**Whisk2CapCut EN**:
```
BEFORE:
- 200 AI images sitting in a folder
- 3 days of dragging, renaming, subtitle-matching
- burnout before the video was even half done

WHAT I DID:
- built a desktop app that reads my scene data
- auto-downloads images from Google Whisk
- generates a complete CapCut project file

AFTER:
- click one button
- 10 minutes later: full CapCut project ready
- energy saved for the part that actually matters — editing
```

**Whisk2CapCut KO**:
```
BEFORE:
- AI 이미지 200장이 폴더에 쌓여 있음
- 3일 동안 끌어다 놓기, 이름 바꾸기, 자막 맞추기
- 영상 반도 안 됐는데 번아웃

WHAT I DID:
- 씬 데이터를 읽는 데스크톱 앱을 만듦
- Google Whisk에서 이미지 자동 다운로드
- 완전한 CapCut 프로젝트 파일 자동 생성

AFTER:
- 버튼 하나 클릭
- 10분 후: CapCut 프로젝트 완성
- 진짜 중요한 편집에 에너지를 쓸 수 있게 됨
```

## 템플릿 선택 가이드

| 상황 | 추천 템플릿 | 이유 |
|------|-----------|------|
| 첫 글 / 자기소개 | Template 1 (Transformations) 또는 5 (압축 가치) | 누군지 + 뭘 하는지 한 번에 전달 |
| 대화 유도가 목표 | Template 2 (Call People Out) 또는 4 (Controversy) | 답글을 끌어냄 |
| 저장/공유 유도 | Template 5 (압축 가치) 또는 6 (Before-Path-After) | 실용적 가치 → bookmark |
| 커뮤니티 공감 | Template 3 (Lessons) | 실패담은 항상 공감 |
| 논쟁/토론 유도 | Template 4 (Controversy) | 양쪽 의견 다 답글 |
