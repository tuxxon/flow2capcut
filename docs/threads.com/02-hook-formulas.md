# Hook Formulas — 훅 공식 + Whisk2CapCut 적용 예제

> 첫 줄이 안 잡히면 나머지는 의미 없다. 스크롤을 멈추게 하는 훅 패턴 정리.

---

## 1. 4단계 훅 공식

```
Bold Statement → Tension → Twist → Credibility(선택)
```

| 단계 | 역할 | 예제 |
|------|------|------|
| **Bold Statement** | 놀라게 / 충격 | "240 AI images. 3 nights. zero automation." |
| **Tension** | 고통/문제 공감 | "downloading one by one. dragging each to the timeline." |
| **Twist** | 반전 | "so i automated the entire thing. now it's one click." |
| **Credibility** | 증거 (선택) | "went from 3 days to 10 minutes." |

---

## 2. 훅 패턴별 공식 + 예제

### 2-1. "I spent..." (신뢰성 + 호기심)

> 구체적 시간/노력을 투자했다는 사실이 신뢰를 만든다.

**공식**: `I spent [구체적 시간] doing [구체적 작업]. Here's what [결과/발견].`

**일반 예제**:
```
I spent 100 hours learning prompt engineering.
Here's what nobody tells you about it.
```

```
I spent 5 years managing social media for Fortune 500 companies.
Here are the 10 things I wish someone had told me on day one.
```

**Whisk2CapCut 적용**:

EN:
```
i spent 3 nights dragging 240 AI images into CapCut one by one.

here's what i learned: this shouldn't be a human's job.
```

KO:
```
AI 이미지 240장을 CapCut에 하나씩 끌어다 놓는 데 3일 밤을 샜어요.

그때 깨달은 게 있거든요 — 이건 사람이 할 일이 아니라는 거.
```

---

### 2-2. "Nobody talks about..." (내부자 지식)

> 모두가 경험하지만 아무도 말하지 않는 것을 지적한다.

**공식**: `Nobody talks about [숨겨진 진실/문제] when it comes to [주제].`

**일반 예제**:
```
nobody talks about the boring part of making AI videos.

not the prompting. not the editing.
the part where you drag 200 images into a timeline. one. by. one.
```

**Whisk2CapCut 적용**:

EN:
```
nobody talks about the most boring part of AI video creation.

it's not writing prompts.
it's not editing.

it's the 6 hours you spend dragging images into CapCut one by one.
```

KO:
```
AI 영상 만들 때 아무도 안 말하는 게 있어요.

프롬프트 작성? 아니요.
편집? 그것도 아니에요.

이미지를 CapCut에 하나씩 끌어다 놓는 6시간이요.
```

---

### 2-3. "I wasted..." (취약성 + 교훈)

> 실패/낭비를 인정하면 진정성이 생기고, 교훈이 가치를 준다.

**공식**: `I wasted [시간/돈] on [잘못된 접근]. Then I [발견/전환점].`

**일반 예제**:
```
I wasted 2 years building features nobody asked for.
Then I realized: talk to users first, build second.
```

**Whisk2CapCut 적용**:

EN:
```
i wasted 3 full days on something a script could do in 10 minutes.

downloading AI images. renaming them. dragging into CapCut.
adding subtitles manually.

that weekend i wrote the first version of an automation tool.
never looked back.
```

KO:
```
스크립트로 10분이면 끝날 일에 3일을 날렸어요.

AI 이미지 다운받고. 이름 바꾸고. CapCut에 끌어다 놓고.
자막을 하나씩 달고.

그 주말에 자동화 도구 첫 버전을 만들었어요.
다시는 수작업 안 해요.
```

---

### 2-4. "Unpopular opinion..." (대담한 의견)

> 통념에 반하는 주장 → 동의/반대 양쪽 다 답글을 단다.

**공식**: `Unpopular opinion: [통념에 반하는 주장]. Here's why: [근거].`

**일반 예제**:
```
unpopular opinion: posting less frequently grows your audience faster than posting daily.
```

**Whisk2CapCut 적용**:

EN:
```
unpopular opinion: most AI video workflows are stuck in 2023.

people still download images one by one.
still drag them into editors manually.
still add subtitles frame by frame.

the tools exist to automate all of this.
most people just don't know about them yet.
```

KO:
```
좀 과격한 말일 수 있는데요 — 대부분의 AI 영상 워크플로우는 2023년에 멈춰있어요.

아직도 이미지 하나씩 다운받고.
편집기에 수동으로 끌어다 놓고.
자막을 프레임별로 달고.

이걸 전부 자동화하는 도구가 있는데
대부분 아직 모르더라고요.
```

---

### 2-5. "Quick question..." (질문형)

> 답변을 유도하므로 답글 수 증가 → 알고리즘 최대 부스트.

**공식**: `[대상]에게 진짜 궁금한 건데 — [구체적 질문]?`

**일반 예제**:
```
quick question for content creators —
how much time do you spend on the boring repetitive stuff vs actual creating?
```

**Whisk2CapCut 적용**:

EN:
```
quick question for AI video creators —

how long does it take you to put 200 images into a CapCut project?

downloading one by one.
dragging each to the timeline.
adding subtitles.
adding animations.

it took me 3 all-nighters.
```

KO:
```
AI 영상 만드는 분들한테 진심으로 궁금한 건데요 —

이미지 200장을 CapCut 프로젝트에 넣는 데 얼마나 걸려요?

하나씩 다운받고.
타임라인에 하나씩 끌어다 놓고.
자막 달고.
애니메이션 넣고.

저는 3일 밤을 샜거든요.
```

---

### 2-6. "X years → X minutes" (압축 가치)

> "내 경험을 압축해서 줄게" → 저장(bookmark) 유도 → 알고리즘 부스트.

**공식**: `Took me [긴 시간] to figure this out. You get it in [짧은 시간].`

**Whisk2CapCut 적용**:

EN:
```
took me 6 months to build this workflow.
you get it in 2 minutes:

my AI video pipeline:
1. write script
2. TTS → voice + SRT
3. split scenes from SRT
4. image prompts per scene
5. bulk generate with Google Whisk
6. one-click export to CapCut
7. final edit → publish

steps 5-6 used to take 3 days. now: 10 minutes.
```

KO:
```
이 워크플로우 만드는 데 6개월 걸렸어요.
2분이면 다 알려드릴게요:

나의 AI 영상 파이프라인:
1. 대본 작성
2. TTS → 음성 + SRT 자막
3. SRT 기반 씬 분할
4. 씬별 이미지 프롬프트
5. Google Whisk로 대량 생성
6. CapCut 프로젝트로 원클릭 내보내기
7. 최종 편집 → 업로드

5~6번, 원래 3일 → 이제 10분.
```

## 3. 훅 작성 핵심 수치

| 규칙 | 데이터 |
|------|--------|
| 구체적 수치 훅 vs 모호한 훅 | **340% 더 높은 성과** |
| 첫 100자에 훅 | 초기 조회수 **+40%** |
| "I" 1인칭 시작 | 3인칭 대비 참여율 **+23%** |
| 250자 이내 | 짧을수록 공유 ↑ (200자 미만 최적) |

---

## 4. 훅 체크리스트

글 올리기 전에 체크:

- [ ] 첫 줄에 **구체적 숫자**가 있는가? (240장, 3일, 10분)
- [ ] **호기심 갭**이 있는가? (결과를 바로 안 알려주고 궁금하게)
- [ ] **1인칭**으로 시작하는가? ("I spent..." / "저는...")
- [ ] **100자 이내**에 훅이 완성되는가?
- [ ] 스크롤 하다가 **멈출 만한 문장**인가?
